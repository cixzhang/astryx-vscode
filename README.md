# Astryx VS Code Theme

> **⚠️ WIP — Not yet tested inside VS Code.**
> The component preview renders in a headless browser, but has not been tested as a running VS Code webview extension yet.

An Astryx theme package that derives from your active VS Code color theme. Built the same way as `@astryxdesign/theme-neutral` — a `defineTheme()` source file compiled by `astryx theme build` into CSS + JS artifacts.

## How It Works

### Theme source: `src/vscodeTheme.ts`

The theme source follows the exact same structure as [neutralTheme.ts](https://github.com/facebook/astryx/blob/main/packages/theme-neutral/src/neutralTheme.ts):

```ts
import {defineTheme} from '@astryxdesign/core/theme';
import {neutralTheme} from '@astryxdesign/theme-neutral/built';

export const vscodeTheme = defineTheme({
  name: 'vscode',
  extends: neutralTheme,

  tokens: {
    '--color-background-surface': 'var(--vscode-editor-background, #0a0a0a)',
    '--color-text-primary':        'var(--vscode-editor-foreground, #fafafa)',
    '--color-accent':              'var(--vscode-button-background, #0e639c)',
    '--color-syntax-keyword':      'var(--vscode-symbolKeyword-foreground, ...)',
    // ... 170 token overrides
  },
});
```

The key difference from other Astryx themes: token values are `var(--vscode-*)` CSS variable references instead of `['#light', '#dark']` hex arrays. CSS resolves these synchronously against the `--vscode-*` custom properties that VS Code injects onto every webview's `<body>`. No `getComputedStyle`, no flash, no runtime color extraction.

### Built artifacts: `dist/`

Generated by the Astryx CLI:

```bash
npx astryx theme build src/vscodeTheme.ts -o dist/theme.css
```

Produces:
- `dist/theme.css` — scoped CSS with 170 token overrides (same format as `@astryxdesign/theme-neutral/theme.css`)
- `dist/vscode.js` — built JS theme object (for `import { vscodeTheme }`)
- `dist/vscode.d.ts` — TypeScript types

### Usage in a VS Code webview

```html
<!-- Load Astryx base + vscode theme CSS -->
<link rel="stylesheet" href="https://unpkg.com/@astryxdesign/core@0.1.2/reset.css">
<link rel="stylesheet" href="https://unpkg.com/@astryxdesign/core@0.1.2/astryx.css">
<link rel="stylesheet" href="./dist/theme.css">
```

```tsx
import { vscodeTheme } from './dist/vscode';
import { Theme } from '@astryxdesign/core/theme';

<Theme theme={vscodeTheme} mode="dark">
  <Button label="Primary" variant="primary" />
</Theme>
```

When the VS Code theme changes, `--vscode-*` variables update and all components re-paint via CSS cascade. The extension listens for `onDidChangeActiveColorTheme` to toggle light/dark mode on the `<Theme>` provider.

### Token mapping

| VS Code Variable | Astryx Token | Fallback |
|---|---|---|
| `--vscode-editor-background` | `--color-background-surface` | `#0a0a0a` |
| `--vscode-editor-foreground` | `--color-text-primary` | `#fafafa` |
| `--vscode-button-background` | `--color-accent` | `#0e639c` |
| `--vscode-sideBar-background` | `--color-background-body` | `#1b1b1b` |
| `--vscode-descriptionForeground` | `--color-text-secondary` | `#a3a3a3` |
| `--vscode-errorForeground` | `--color-error` | `#f48771` |
| `--vscode-symbolKeyword-foreground` | `--color-syntax-keyword` | editor fg |
| `--vscode-symbolString-foreground` | `--color-syntax-string` | editor fg |
| `--vscode-editorComment-foreground` | `--color-syntax-comment` | desc fg |
| ... | ... | 170 total |

### How VS Code exposes theme colors

VS Code provides theme colors to webviews via CSS variables injected on `<body>`. There is no extension host API to read actual color values — `vscode.window.activeColorTheme` only exposes `kind` (Light/Dark/HighContrast). The `--vscode-*` CSS variables are the only channel for raw color values.

## VS Code Extension

The extension provides a command that opens a webview showing live Astryx components themed by the active VS Code color:

1. Open Command Palette (`Cmd+Shift+P`)
2. Run `Astryx: Show Component Preview`

## Development

```bash
npm install

# Build the theme (regenerates dist/theme.css, dist/vscode.js)
npx astryx theme build src/vscodeTheme.ts -o dist/theme.css

# Compile the extension
npm run compile

# Package as VSIX
npx @vscode/vsce package --no-yarn

# Install
code --install-extension astryx-vscode-0.3.0.vsix

# Or launch Extension Development Host
# Press F5 in VS Code
```
